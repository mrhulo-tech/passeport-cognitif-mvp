# MVP Demo Checklist

## Before the demo
- Vérifier que le déploiement Vercel est en statut Ready
- Vérifier que le backend Render répond
- Vérifier l’utilisateur 1
- Vérifier l’utilisateur 2
- Vérifier que les recommandations s’affichent
- Vérifier que les métriques s’affichent
- Vérifier que la navigation utilisateur fonctionne

## URLs
- Frontend principal
- `/?user=1`
- `/?user=2`
- Backend `/dashboard/1`
- Backend `/dashboard/2`

## Key messages
- profil apprenant dynamique
- données observables
- indicateurs interprétables
- recommandations personnalisées
- aucune mesure objective de l’intelligence
- valeur démontrable dans le LMS

## Risk checks
- ne pas ouvrir une vieille URL Vercel
- ne pas utiliser un ancien clone local comme source de vérité
- ne pas modifier les fichiers de build juste avant la démo
- vérifier que les derniers commits sont bien déployés

## If something fails
- garder le message centré sur la démonstration du concept MVP
- montrer au minimum la variation utilisateur 1 / utilisateur 2
- insister sur la logique pédagogique et la lisibilité
